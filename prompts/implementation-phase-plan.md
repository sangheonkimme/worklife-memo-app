# 워크라이프 메모 앱 구현 Phase 체크리스트

> 참고 문서: `prompts/memo-app-plan.md`  
> 주요 타깃: 대한민국 1,000만 직장인  
> 각 Phase 완료 시 체크박스를 업데이트하고 다음 단계 진입 여부를 점검하세요.

## 전반 준비
- [ ] 타깃 사용자(대한민국 직장인 1,000만 명)의 페인 포인트와 업무 맥락 정리
- [ ] MVP 범위와 스토리맵 공유를 위한 협업 채널 세팅

## Phase 0. 비전 및 범위 확정 (Week 0)
- [ ] 제품 비전과 MVP 범위 합의
- [ ] 핵심 사용자 페르소나와 메모 유형 문서화
- [ ] 오프라인 우선 요구사항 및 기대치 정의
- [ ] 리마인더·검색·협업 기능 우선순위 확정
- [ ] 일일 활성 메모 수, 동기화 지연, Crash-Free 비율 등 KPI 합의
- [ ] 문제/솔루션 브리프, 우선순위 리스트, KPI 정의서 리뷰 완료

## Phase 1. UX 및 브랜딩 탐색 (Weeks 1-2)
- [ ] 메모 목록/편집기/조직/검색/설정 사용자 플로우 설계
- [ ] 로우파이 프로토타입 제작
- [ ] 하이파이 프로토타입 완성
- [ ] 사용자 테스트(3~5명) 진행 및 인사이트 정리
- [ ] 디자인 시스템 토큰 및 접근성 가이드 수립
- [ ] 브랜드 톤&매너 정리
- [ ] 사용자 여정 지도, 프로토타입, 테스트 메모 공유

## Phase 2. 기술 기반 및 환경 세팅 (Weeks 2-3)
- [ ] Flutter + Firebase/Firestore + Firebase Auth + Secure Storage + Hive/SQLite 스택 확정
- [ ] 레포지토리 구조 및 브랜치 전략 수립
- [ ] GitHub Actions 기반 CI 파이프라인 구현
- [ ] 린트/포맷/테스트 자동화 적용
- [ ] 분석(Firebase Analytics) 및 크래시 리포팅(Crashlytics) 연동
- [ ] Firestore 보안 규칙 초안 작성
- [ ] 로컬 스토리지 암호화 정책 수립
- [ ] GDPR/CCPA 체크리스트 작성
- [ ] 개발 가이드 및 환경 구성 문서화

## Phase 3. 코어 기능 빌드업 I (Weeks 3-6)
- [ ] 스프린트 1 킥오프 및 작업 배정
- [ ] 인증 및 온보딩 플로우 구현
- [ ] 오프라인 지속성을 갖춘 로컬 메모 CRUD 개발
- [ ] 핵심 단위 테스트 작성 및 통과
- [ ] 스프린트 1 데모 및 회고 완료
- [ ] 스프린트 2 킥오프 및 동기화 요구사항 확인
- [ ] 실시간 동기화(충돌 정책 포함) 구현
- [ ] 기본 검색/필터 기능 추가
- [ ] 통합 테스트 작성 및 통과
- [ ] 동기화 충돌 텔레메트리 설계 및 계측
- [ ] 알림 신뢰성 검증 계획 수립
- [ ] 스프린트 2 데모 및 회고 완료
- [ ] 핵심 기능 릴리즈 브랜치 생성 및 코드 리뷰 완료

## Phase 4. 코어 기능 빌드업 II (Weeks 7-10)
- [ ] 스프린트 3 킥오프 및 목표 정렬
- [ ] 리마인더/알림 기능 구현
- [ ] 첨부(이미지/오디오) 기능 구현
- [ ] 라이트/다크 테마 지원 완료
- [ ] 디바이스 팜 기반 E2E 테스트 파이프라인 구성
- [ ] 스프린트 3 데모 및 회고 완료
- [ ] 스프린트 4 킥오프 및 성능 목표 확인
- [ ] 공유/내보내기(파일, 링크) 기능 구현
- [ ] 설정(백업·데이터 내보내기) 추가
- [ ] 성능 모니터링 및 주요 시나리오 튜닝
- [ ] 분석 대시보드 초기 버전 구성
- [ ] Firebase Test Lab 야간 자동 실행 및 리포트 검토
- [ ] 피처 플래그 전략 운영 및 롤아웃 계획 수립
- [ ] 스프린트 4 데모 및 회고 완료
- [ ] 고급 기능 배포 후보 빌드 승인

## Phase 5. 품질 보증, 출시 및 운영 (Weeks 10-12+)
- [ ] iOS/Android 폼 팩터별 회귀 테스트 진행
- [ ] 탐색 테스트 및 버그 트리아지 완료
- [ ] 접근성 감사 수행 및 개선 반영
- [ ] 현지화 프레임워크 적용(단일 언어 대비 API 준비)
- [ ] App Store Connect/Play Console 메타데이터 작성
- [ ] 개인정보 처리방침 및 이용 약관 정비
- [ ] 베타 배포(TestFlight, Play 내부 테스트) 시행 및 피드백 수집
- [ ] 단계적 롤아웃 전략 수립 및 모니터링 대시보드 구축
- [ ] 로그/알림 체계 운영 시작
- [ ] 고객 지원 워크플로우 및 FAQ 정리
- [ ] 향후 백로그(협업 메모, 위젯, 데스크톱/웹 앱) 우선순위 재평가
- [ ] 운영 지표 기반 개선 사이클 정의
- [ ] 출시 후 회고 및 로드맵 업데이트
